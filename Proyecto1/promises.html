Promises

(goodbye callback hell)

Descripcion

Una promesa o promise se usa para computacion asincrona. Puede estar definida en tres estados: pendiente, cumplida o rechazada, dependiendo si la operacion aun no se realiza, se realizo satisfactoriamente o fue fallida, respectivamente.

Caracteristicas

  • 1. El metodo devuelve una promesa de tener un valor en algun momento en el futuro.
  • .2 Sintaxis: new Promise(ejecutor) o new Promise(function(resolver, rechazar){...});
  • 3. Una promesa pendiente puede llegar a cumplirse con un valor o ser rechazada con una razon. En caso de que cualquier sucede, procede a ejecutar lo definido en then.
  • 4. Una promesa permite definir funciones que retornan valores de forma asincrona.

Ejemplo descriptivo

                    
//Se define la promesa
fs.readFile('config.txt',
        function (error, text) {
           //En caso de error ingresa en la primera parte
            if (error) {
                console.error('Error al leer archivo');
            } else {
               //Si no hay error en la funcion intenta realizar la operacion 
               //Se utiliza try-catch para capturar problemas
                try {
                    console.error('Archivo leyendose');
                } catch (e) {
                    console.error('Archivo incorrecto');
                }
            }
        });
               
//Luego
//En caso de que ya se haya ejecuta el if o else anterior se procede a realizar 
//lo siguiente:

readFilePromisified('config.txt')
.then(function (text) { // Realiza las operaciones si no hay error
    console.log('Lectura exitosa de archivo')
})
.catch(function (reason) { // Si se dan errores
    // En este caso, si algun caracter o hay un problema en el archivo que se 
    // esta leyendo
    console.error('Surgio un error', reason);
});
